Calendar Synchronization Scenarios

The following are some common scenarios to help explain how calendar synchronization works.

 Synchronization is affected by the built-in data security associated with the user who is logged on to Saleslogix. The amount of data available to transfer is limited by the security rights of the user logged in.

Synchronization Scenarios

Activity types in Saleslogix may synchronize to different activity types in an integrated application. See the Understanding Calendar Data Exchange table for details.

For the following scenarios:

  • "calendar item" will be used to represent a calendar activity type in an integrated application, and phone calls, meetings, or personal activities in Saleslogix.
  • "task" will be used to represent a non-calendar activity type in an integrated application and a to-do in Saleslogix.
  • "either application" refers to Saleslogix and the integrated application.
Action Result
Scheduling, Editing, Completing or Deleting Calendar Items
Schedule a calendar item in either application. The calendar item synchronizes.
Edit a calendar item in either application. The calendar item change synchronizes.
Edit the same calendar item in both applications during the same synchronization cycle.

The calendar item that was edited most recently will synchronize to the other application and update all values with any differences.

Complete a calendar item in Saleslogix. The calendar item is completed in Saleslogix and the word "Completed" appears in the Subject line of the item in the integrated application.
Delete a calendar item in either application. The calendar item is removed from the user's calendars in both applications.

Scheduling, Editing, Completing or Deleting Tasks
Schedule a task in either application. The task synchronizes.
Edit a task in either application. The task change synchronizes.
Edit the same task in both applications during the same synchronization cycle.

The task that was edited most recently will synchronize to the other application and update all values with any differences.

Complete a to-do in Saleslogix. The change synchronizes and in the task is marked as completed in the integrated application.
Complete a task in an integrated application. The change synchronizes and the to-do is completed and logged as a history item in Saleslogix.
Schedule a task that has no start date or time in an integrated application. The task synchronizes as a to-do in Saleslogix with the create date as the start date. During the next synchronization the new start date will synchronize and update the task in the integrated application.

Scheduling Calendar Items and Tasks with Multiple Occurrences
Schedule a recurring calendar item in Saleslogix. The calendar item and all occurrences are synchronized to the integrated application.
Schedule a calendar item with multiple occurrences in the integrated application.

The calendar item is synchronized to Saleslogix with the maximum number of occurrences allowed in Saleslogix:

  • Daily - 999
  • Weekly - 260
  • Monthly - 60
  • Yearly - 20

This limit is synchronized back to the integrated application.
Schedule a recurring to-do in Saleslogix. The recurring to-do is synchronized to a recurring task in the integrated application.
Schedule a recurring task in an integrated application.

The task is synchronized to Saleslogix as a to-do with the maximum number of occurrences allowed in Saleslogix:

  • Daily - 999
  • Weekly - 260
  • Monthly - 60
  • Yearly - 20

This limit is synchronized back to the integrated application.

Scheduling Calendar Items with Members
Schedule a calendar item in either application, where I am the leader and there are 3 other members.

In the application where the calendar item was scheduled, the calendar item includes the leader and member information as scheduled. When it synchronizes with the other application, you remain the leader, but it does not contain any member information.

The member information will remain in calendar item of the originating application, even if the calendar item is updated.
Schedule a calendar item in either application, where I am a member, not the leader.

In the application where the calendar item was scheduled, the calendar item includes the leader and member information as scheduled. When it synchronizes with the other application, you become the leader, and it does not contain any member information.

The member information will remain in calendar item of the originating application, even if the calendar item is updated.
Change the leader from me to another user in Saleslogix after the calendar item has synchronized.

Deletes the calendar item in an integrated application.*

*You cannot change the leader of a calendar item in Outlook.

Accepting and Declining Calendar Item Invitations
Accept a calendar item invitation in either application before the calendar item is synchronized. The calendar item is synchronized and indicates that it is accepted.
Accept a calendar item invitation in either application after the calendar item is synchronized. The calendar item synchronizes when it is first scheduled and again after it is updated to indicate that is accepted.
Decline a calendar item invitation in either application before the calendar item is synchronized. The calendar item is not synchronized to the other application.
Decline a calendar item invitation in either application after the calendar item is synchronized. The calendar item is removed from the user's calendars in both applications.

Working with Personal or Private Calendar Items
Schedule a personal activity in Saleslogix. The personal activity synchronizes as a calendar item set as Private in the integrated application.
Schedule a private calendar item in an integrated application. The private calendar item synchronizes as a personal activity in Saleslogix.
In an integrated application, clear the Private flag after the calendar item has been synchronized. The calendar item synchronizes and changes the personal activity to a meeting in Saleslogix.
In an integrated application, select the Private flag after the calendar item has been synchronized. The calendar item synchronizes and changes the meeting to a personal activity in Saleslogix.

Working with Timeless Calendar Items and Tasks
Schedule a timeless calendar item in Saleslogix. The timeless calendar item synchronizes as an all day event to the integrated application.
Schedule an all day event in an integrated application. The all day event synchronizes as a timeless meeting in Saleslogix.
Change a calendar item to timeless in Saleslogix. The calendar item is changed to an all day event in the integrated application.
Roll over a timeless calendar item in Saleslogix. The change synchronizes to the integrated application and changes the start date to the current, rolled over date.

Snoozing Alarms
Snooze an alarm for a calendar item in either application.

The calendar item change synchronizes.

Depending on when the calendar item is snoozed during the synchronization cycle the alarm may be triggered before the "snooze" is synchronized.

Saleslogix recommends snoozing alarms in only one of the applications.
Snooze an alarm for a task in either application.

The task change synchronizes.

Depending on when the calendar item is snoozed during the synchronization cycle the alarm may be triggered before the "snooze" is synchronized.

Saleslogix recommends snoozing alarms in only one of the applications.
