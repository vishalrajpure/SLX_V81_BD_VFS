Understanding Calendar Integration

Calendar integration allows you to synchronize your calendar between Saleslogix and a supported integration. Your administrator must configure and grant permission before you can synchronize calendars with an integration.

During a synchronization cycle, Saleslogix compares fields in both systems, and makes changes to ensure both systems contain the same information. Only data in mapped fields between the applications is transferred. The data is compared and changes are made to ensure the information matches. For example, if a record has been added to one system and not to the other, that record is added to the incomplete system.

 Synchronization is affected by the built-in data security associated with the user who is logged on to Saleslogix. The amount of data available to transfer is limited by the security rights of the user logged in.

Calendar Sync Behavior

  • If the same record is updated in both systems during the same sync cycle, the latest change wins.
  • Deleting an activity in either system deletes the activity in both application.
  • Activity leaders members and resources:
    • Member and resource information is not synchronized.
    • If the user is a member or a leader of an activity, when the activity is synchronized the user will be listed as the leader.
  • Unconfirmed and confirmed activities will sync.
  • Declined activities:
    • Activities that are declined before a synchronization cycle are not synchronized.
    • Activities that are declined after a synchronization cycle are removed from the user's calendar in both applications.
  • Activity types in Saleslogix may synchronize to different activity types in an integrated application. See the Understanding Calendar Data Exchange table for details.

For specific calendar integration scenarios, see Calendar Synchronization scenarios.

Understanding Calendar Data Exchange

The following table lists how Saleslogix calendar data is synchronized with external integrations:

Saleslogix Syncs As Outlook
Meetings syncs as Appointments
Phone Calls syncs as Appointments
Personal Activites syncs as Appointments, flagged as Private
To-Dos syncs as Tasks
Events syncs as do not synchronize
Timeless Phone Call, Meeting or Personal Activity syncs as All day event

How Do I?

  • Set calendar options
  • Manually Synchronizing Calendar and Contacts

Simultaneously synchronizing the Remote Client to the main office database and Microsoft Outlook, is not recommended. Doing so significantly reduces synchronization speed and causes memory problems. Saleslogix recommends that you synchronize with Outlook before or after synchronizing your remote database.
