Contact Synchronization Scenarios

The following are some common scenarios to help explain how contact synchronization works.

Synchronization is restricted by the built-in data security associated with the user who is logged on to Saleslogix. The amount of data available to transfer is limited by the security rights of the user logged in.

Stop synchronizing a contact.

In the integrated application, move the contact to a contact folder other than the contact sync folder and run a synchronization cycle.

 The contact is removed from the contact sync group, so that it is no longer synchronized to the integrated application, but is not deleted from Saleslogix.
Delete a contact from Saleslogix from both applications. Delete the contact in Saleslogix, and run a synchronization cycle. The contact is deleted in Saleslogix and moved to the Deleted Items folder where it can be permanently deleted.
Delete a contact from Saleslogix, but keep the contact in integrated application. Delete the contact in Saleslogix, and run a synchronization cycle. Open the Deleted Items folder, find the contact, and move the contact to a contact folder other than the contact sync folder. The contact is deleted in Saleslogix, but still available in the integrated application.
